i. 운영 서버 배포 형태

# 1-1. 운영 서비스 구성 {수정 금지}
	- 외부 HTTP 요청을 Nginx가 받는다.
	- Nginx가 loopback의 Tomcat 8080으로 KMPC 요청을 전달한다.
	- Tomcat의 ROOT webapp이 실제 KMPC 애플리케이션이다.


# 1-2. 기존에 알려진 배포 흐름 {수정 금지}
	1. GitHub에서 승인·병합된 dev 커밋을 확정한다.
	2. 서버의 source 또는 별도 release worktree에서 해당 SHA를 준비한다.
	3. 운영 환경용 environment.properties와 file store는 Git 소스와 분리해 유지한다.
	4. Maven으로 전체 릴리스 빌드를 한다.
	5. 기존 /opt/tomcat9/webapps/ROOT와 필요한 Nginx 설정을 백업한다.
	6. 새 빌드 결과를 ROOT에 배포한다.
	7. Tomcat과 필요한 서비스를 재시작한다.
